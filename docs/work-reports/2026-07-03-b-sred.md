## SR&ED Project 1 (partial): Cohort-state migration without corrupting a live experiment's attribution

*Mapped from: Task 1 in work report 2026-07-03-b.md*

### 1. Technological Uncertainty (Line 242)
Retiring one arm of a running, cookie-persisted A/B/C/D experiment mid-flight raised a data-integrity question: how to reassign the retired cohort's returning visitors and in-flight traffic without producing mislabeled conversions or double-counting, given that assignment state lives in three places that update at different times (a 30-day client cookie, a request-scoped SSR header, and the lead payload's `experiment_variant`).

### 2. Systematic Investigation (Line 244)
Traced the three assignment surfaces; determined the safe migration is to (a) rewrite a stale `A` cookie to `B` at the edge *and* re-persist it so subsequent views are consistent, (b) default the SSR resolution to `B` so a first-view/no-header case can't render a retired arm, and (c) leave the payload attribution key untouched so historical A conversions remain distinguishable in the tracker. Verified live that `?ab=` overrides + fresh assignments set the cookie and header consistently and that the dashboard retains A's historical rows at 0% allocation.

### 3. Technological Advancement (Line 246)
Established a repeatable pattern for retiring/reallocating an experiment arm across a multi-surface (cookie + SSR header + payload) assignment system without losing attribution continuity — reusable for future variant consolidations on this and other LPs.
