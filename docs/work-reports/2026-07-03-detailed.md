## Task 6: First-party variant tracker

### `scripts/track-server.mjs` (VM, PM2 `88w-track`)
- `node:http` only. Loads `REPO_ROOT/.env` for `TRACK_STATS_TOKEN`. Config: `TRACK_PORT=8088`, `TRACK_DATA=/home/amir/88w-track/events.jsonl`, `TRACK_ORIGIN=https://go.88westrealty.com`.
- `POST /event` — validates `v∈{A,B,C,D}`, `e∈{view,lead}`, `sid` string; appends `{v,e,sid,s,ts}` JSONL; 204. Junk silently ignored.
- `GET /stats?token=` — aggregates distinct-session views/leads + raw counts + CVR per variant.
- `GET /dashboard?token=` — HTML: KPIs, per-variant table (design, configured `SPLIT` vs observed session share, sessions, leads, CVR, ★ best ≥5 sessions), clickable live LP URL + per-variant `?ab=X` links, 60s auto-refresh.
- CORS for `TRACK_ORIGIN`; `OPTIONS` → 204.

### Client — `lib/track.ts`
- `trackEvent("view"|"lead", source?)` reads `_lp_ab_cohort` cookie (variant) + a random `sessionStorage._lp_sid`. "view" fires once/session (`_lp_viewed` guard). Sends via `navigator.sendBeacon` with `Blob type:"text/plain"` (CORS-safe → no preflight); server JSON-parses. Fired in `ForeclosureLeadProvider` mount (view) + `ForeclosureLeadForm` submit success (lead).

### Infra
- `ecosystem.config.cjs`: PM2 app `88w-track` (autorestart). `pm2 start … --only 88w-track && pm2 save`.
