# Deal data — how to update the deal cards on the LPs

The deal cards on `/foreclosure-deals-vancouver` and
`/developer-condo-deals-vancouver` are driven by JSON files in this
directory. Edit the JSON, commit, deploy — the cards update on the next
Vercel build.

## Files

| File | Drives | How updated |
|---|---|---|
| `foreclosure-deals.json` | The 6 cards on `/foreclosure-deals-vancouver` | Manual only (MLS data is licensed; we do not auto-scrape) |
| `developer-deals.json` | The 6 tiles on `/developer-condo-deals-vancouver` | Manual, optionally seeded from `developer-deals.scraped.json` |
| `developer-deals.scraped.json` | **Internal research artifact** — never read by the page | Auto-generated by `npm run scrape:deals` (pulls from Mike Stewart's public incentive page) |

## Phase 1 workflow — manual update (do this every week)

1. Open the JSON file you want to update (`foreclosure-deals.json` or `developer-deals.json`).
2. Edit the `deals[]` array. Keep exactly **6 entries** — the layout assumes 6 tiles.
3. Update `lastUpdated` to today's date (`YYYY-MM-DD`).
4. Commit and push. Vercel rebuilds and deploys.

### `foreclosure-deals.json` schema

```json
{
  "type": "Detached | Townhouse | Condo | Half-duplex",
  "area": "Vancouver (Hastings-Sunrise)",
  "street": "████ E 26th Ave, Vancouver",
  "beds": "4 bed",
  "baths": "2 bath",
  "sqft": "1,820 sqft",
  "listed": "$1.28M",
  "assessed": "$1.42M",
  "courtDate": "Jul 15",
  "daysOnMarket": 142,
  "tag": "Schedule A required | Vacant — easy showings | Tenanted — 60-day notice | etc.",
  "image": "/images/shared/properties/<one-of-the-existing-images>.jpg",
  "imageAlt": "..."
}
```

Mask the street number with `████` until subscriber sign-up — that's
intentional. Court files are public, but we don't run a billboard.

Available property images in `/images/shared/properties/`:
- `condo-modern-glass.jpg`
- `condo-minimalist.jpg`
- `condo-courtyard.jpg`
- `townhouse-brick.jpg`
- `detached-craftsman.jpg`
- `detached-suburban.jpg`

## Phase 2 workflow — seeding from Mike Stewart's public page

Mike Stewart maintains a daily-updated grid of ~250 active Vancouver
pre-sale projects with their published incentives at
[`mikestewart.ca/vancouver-presale-condo-incentives/`](https://www.mikestewart.ca/vancouver-presale-condo-incentives/).
We scrape only the **published marketing teasers** (not MLS listings,
not photos) into `developer-deals.scraped.json` as a starting research
artifact. The realtor still curates 6 entries by hand into the live
`developer-deals.json`.

### Run the scraper

```bash
npm run scrape:deals
```

This writes `data/developer-deals.scraped.json` with up to 30 projects.
Each entry has:

```json
{
  "project_name": "Italia by Solterra – 2323 Fir Street",
  "city": "Vancouver",
  "incentive_summary": "For Monaco project: A few final units remain... (verbatim teaser, truncated at ~320 chars)",
  "project_url": "https://www.mikestewart.ca/presale/solterra-italia/",
  "headline_dollar": "$100,000"
}
```

### Curate into the live file

The scraped data is **partial** — it doesn't have `stage`, `unitsLeft`,
`expires`, or `image`. Use it as a research starting point:

1. Pick the 6 most relevant projects from `developer-deals.scraped.json`
   (use `headline_dollar` and `city` to filter).
2. Open the project page (`project_url`) for each pick to confirm the
   incentive details, project stage, and unit count.
3. Map fields manually into `developer-deals.json`:
   - `name` ← `project_name` (often shorten)
   - `area` ← `city` + neighborhood (look up on the project page)
   - `total` ← `headline_dollar` or your own better-negotiated number
   - `concessions` ← split `incentive_summary` into 3-5 bullets
   - `stage`, `unitsLeft`, `expires` ← from the project page or your CRM
   - `image` ← cycle through the 6 shared property images
4. Save the file with `"source": "manual"` (or `"source": "scraped+curated"`
   if you want to track that an entry started from a Mike Stewart scrape).
5. Update `lastUpdated`, commit, push.

### Why we don't auto-replace `developer-deals.json` from the scraper

- The scraper pulls competitors' marketing teasers verbatim. Republishing
  them as 88 West Realty's curated deal list without manual review would
  be misleading and a CREA Code-of-Ethics issue.
- Many entries in the scrape are *qualitative* ("Brand new homes ready
  for move-in") with no dollar headline — not useful as tiles.
- The page is a marketing surface, not a data feed. Hand-curated keeps
  quality high.

The right way to think about this: the scraper saves the realtor 30
minutes of opening 18 tabs. They still pick the 6 best, fill in the
fields, and ship.

## Legal & compliance

- **Mike Stewart scraper:** pulls only the published incentive teaser
  text + project name + project URL from a public marketing page. It
  does NOT pull MLS listing data, photos, full project descriptions,
  or any IDX content. Republishing this on our LP without curation
  could attract a complaint, so we always hand-curate before shipping.
- **Foreclosure listings:** are MLS-licensed. We never scrape them from
  competitors. The masked addresses in `foreclosure-deals.json` are
  illustrative samples; real listings only show after subscriber sign-up
  via the 88 West specialist (who pulls from the brokerage's IDX feed).
- **User-Agent:** the scraper identifies itself with a custom UA so
  Mike Stewart's web team can see who's hitting them. If we ever get a
  cease-and-desist, we stop.
